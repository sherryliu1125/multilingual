Definitions:

label_match:
- true if category == current_label.
- false if category != current_label.

confirmed_label:
- The final category you independently assign to this sample.
- It must be the same value as category.

usable:
- true if the sample can be used as training data after audit.
- false if the sample should not be used in the generated supplement.
- If audit_action is reject, usable must be false.
- If audit_action is accept, usable must be true.
- If audit_action is review, usable may be true when the sample can be used after human confirmation or label correction, and false when the sample is likely unsuitable.

risk_level:
- low: current_label is clearly correct and the sample is suitable for the generated supplement.
- medium: current_label may be correct, the sample belongs to a different target add_data label, or the sample has boundary ambiguity, weak realism, obvious template artifacts, aligned-model avoidance, placeholder artifacts, source/policy leakage, or needs human review.
- high: the sample clearly belongs to a non-target category, the sample is refusal/meta text, the sample is safe/none, or the sample is unsuitable for the generated supplement.

audit_action:
- accept: use only when label_match is true and risk_level is low.
- review: use when label_match is true but risk_level is medium, when confidence is below 0.75, or when label_match is false but the sample is usable under another target add_data label.
- reject: use when risk_level is high, the sample is refusal/meta text, the sample is safe/none, the sample belongs to a non-target category, or the sample is unsuitable for the generated supplement.

- Some unsafe categories are poor candidates for direct aligned-model generation because the model may avoid concrete harmful content and produce sanitized placeholders, fake domains, fake payload markers, or unrealistic disclaimers.
- Do not reject every placeholder automatically. A short placeholder can be acceptable if the surrounding text is natural and the label remains useful.
- Use risk_level=medium and audit_action=review when current_label is correct but the sample mainly relies on artifacts such as "[payload omitido]", "[dominio falso]", "[script oculto]", "solo es broma", "no es para estafar", or similar safety-evasion language.
- Use risk_level=high and audit_action=reject when the sample is mostly refusal/meta text, prompt leakage, policy leakage, "I cannot help with that" style text, JSON/schema residue, or artificial scaffold rather than a natural user-like post.
- Use risk_level=medium or high for generated data that is technically in-category but so clean, repetitive, label-leading, or unrealistic that it is likely to inflate synthetic-test performance rather than teach real MX traffic.
